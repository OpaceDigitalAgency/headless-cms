# Payload Front-end Seed Layouts & Block Compositions

> [!NOTE]
> This document defines the *target patterns* for seed data. For the implementation details of how the seeding system works, see the [SEEDING_SYSTEM_GUIDE.md](../SEEDING_SYSTEM_GUIDE.md).

This document defines **seed layouts** for each current content type in your Payload setup.

The aim is to:
- Give editors strong starting points (not restrictive templates)
- Encourage consistent, high‑quality layout patterns
- Maximise SEO, UX, and conversion potential
- Reuse the same layout primitives, components, and Blocks everywhere

---

## 1. Pages (5 common page types)

### 1.1 Home Page

#### Layout A: “Hero‑Led Brand Statement”
**Purpose:** Immediate clarity, brand trust, conversion
- Hero (headline, sub‑headline, primary CTA)
- Logo Cloud (clients/partners)
- Features (3–6 core propositions)
- Testimonials (short quotes)
- CTA (primary conversion)
- FAQ (top objections)

#### Layout B: “Content‑First Editorial Home”
**Purpose:** Thought leadership, publishing‑driven sites
- Hero (minimal, editorial style)
- Content (intro narrative)
- Archive (latest Posts / Case Studies)
- Quote (brand belief)
- CTA (newsletter or contact)

#### Layout C: “Product / Service Hub”
**Purpose:** SaaS or service‑driven sites
- Hero (value proposition + visual)
- Features (benefits)
- Stats (proof points)
- Testimonials
- Pricing (summary tiers)
- CTA

## 4. Notes on SEO & Reuse

- Every layout assumes:
  - Shared Header/Footer
  - Shared SEO generator
  - Structured data per content type
- Blocks never hardcode spacing or width
- Templates define *structure*, Blocks define *content*

These seed layouts should be exposed as **editor presets**, not rigid templates.
